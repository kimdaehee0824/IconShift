# IconShift 기획서

## 한 줄 정의
macOS에서 **시스템 라이트/다크 모드에 맞춰 앱 아이콘을 자동으로 바꿔주는 메뉴바 유틸리티.**

## 문제 정의
- macOS는 앱 번들의 Finder/Dock 아이콘을 `.icns` 하나로 고정한다. 라이트/다크에 따라 앱 아이콘이
  자동으로 바뀌는 기본 메커니즘이 없다.
- 특히 Safari/Chrome 웹앱(PWA)은 밝은 배경 고정 아이콘이 많아, 다크 메뉴바·Dock에서 튀거나
  어두운 배경에 묻힌다.
- 사용자가 매번 수동으로 아이콘을 바꾸는 건 번거롭고, 브라우저 업데이트 시 원복되기까지 한다.

## 타깃 사용자
- 데스크톱 심미성에 민감하고 라이트/다크를 오가는 macOS 파워유저.
- PWA를 앱처럼 쓰는 사람(ChatGPT/Claude/Notion 등 웹앱을 Dock에 두는 사용자).
- 아이콘 커스터마이징을 즐기는 리시(ricing) 커뮤니티.

## 핵심 가치 / 차별점
1. **자동화** — 한 번 지정하면 외형 전환마다 알아서 교체. 수동 개입 0.
2. **앱별 세밀 제어** — 자동전환 / 라이트 고정 / 다크 고정을 앱마다 선택.
3. **안전** — 실패한 아이콘 쓰기가 기존 아이콘을 깨뜨리지 않도록 트랜잭션 롤백.
4. **자가 복구** — 브라우저 업데이트로 리셋된 PWA 아이콘을 앱 시작 시 재적용.
5. **가벼움** — Sparkle(자동 업데이트) 외 의존성 없는 작은 메뉴바 앱. 원하면 아이콘조차 숨기고 백그라운드 상주.

## 기능 범위

### 현재 (v0.1)
- 시스템 외형 감지(KVO + 분산 알림 + 폴링)와 변경 시 자동 적용.
- 앱별 라이트/다크 아이콘 지정(드래그앤드롭·파일 선택), auto/light/dark 모드.
- 설치된 앱 검색 후 추가, 사이드바 검색.
- 메뉴바 아이콘 표시/숨김(숨기면 순수 백그라운드 상주, 재실행 시 창 복귀).
- 로그인 시 실행(SMAppService), 파일 로그, App Management 권한 안내 배너.
- 영어(base)·한국어·일본어 현지화(String Catalog).
- 아이콘 슬롯에서 macOSicons 웹사이트 안내 링크 제공(다운로드는 브라우저에서 수행).
- NavigationSplitView와 통합 툴바를 사용한 macOS HIG 정합 창 구성.
- 일반·정보 탭이 있는 설정 창과 표준 macOS 메인 메뉴.
- Sparkle 자동 업데이트(appcast는 GitHub Release 자산, 메뉴에서 수동 확인 가능).

### 다음 (v0.2 후보)
- 아이콘 미리보기 품질 개선, 여러 해상도 대응.
- 중국어 간체(zh-Hans) 지원 여부 검토.

### 나중 (백로그)
- 아이콘 세트/프리셋 공유, 커뮤니티 아이콘 소스.
- FSEvents로 PWA 번들 재생성 실시간 감지.
- 스케줄/시간 기반 전환(예: 일몰 후 다크 아이콘).
- 아이콘 자동 다크 변형 생성(색 반전/틴팅) 옵션.

## 제품 방향
- **배포**: GitHub 오픈소스(MIT). `1.0.0` 형식 태그에서 GitHub Actions가 Developer ID 서명·노터라이즈된
  Universal DMG를 자동 릴리스 노트와 함께 배포한다.
- **서명**: 개발은 self-signed로 App Management 권한 유지, 공개 배포는 Developer ID + 노터라이즈
  (CI 시크릿 구성은 CONTRIBUTING 참고).
- **접근성/현지화**: 문자열은 String Catalog(.xcstrings)로 관리해 번역 기여를 쉽게.
- **철학**: 작고 명확하게. 설정은 최소, 기본값이 똑똑하게. 사용자의 아이콘/앱을 절대 손상하지 않기.

## 비목표 (Out of scope)
- 앱 아이콘 "제작" 도구(디자인 에디터)는 만들지 않는다 — 지정/적용만.
- 시스템 전역 테마·아이콘 팩 관리자가 아니다.
- iOS/iPadOS 대상 아님(macOS 전용).

## 기술 제약 (요약)
`~/Applications` 등 보호 위치 앱의 아이콘 교체는 **코드서명 + App Management 권한**이 필수이고,
실패 시 롤백이 반드시 필요하며, 실행 중 앱의 Dock 반영은 재실행이 필요하다. 상세는
[CONTRIBUTING](../CONTRIBUTING.md)의 "How it works" 참고.
