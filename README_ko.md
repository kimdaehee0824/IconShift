# IconShift

> [English](README.md), [한국어](README_ko.md), [日本語](README_jp.md)

IconShift는 macOS의 라이트·다크 모드에 맞춰 앱 아이콘을 바꿔 주는 작은 메뉴 막대 앱입니다. 앱마다 라이트용과 다크용 아이콘을 하나씩 지정하면 시스템 외형이 바뀔 때 Finder와 Dock의 아이콘도 알맞게 바뀝니다.

한쪽 배경만 고려해 만들어진 Safari·Chrome 웹 앱 아이콘을 사용하는 경우 특히 유용합니다.

## 주요 기능

- **자동 전환** — 현재 시스템 외형에 맞는 라이트 또는 다크 아이콘을 사용합니다.
- **앱별 설정** — 시스템 외형을 따르거나 항상 라이트·항상 다크 중 하나로 고정할 수 있습니다.
- **간단한 아이콘 지정** — PNG 또는 ICNS 파일을 각 아이콘 칸에 놓으면 됩니다.
- **로그인 시 실행** — 로그인한 뒤 조용히 시작해 아이콘을 계속 맞춰 둡니다.
- **메뉴 막대 아이콘 선택** — 아이콘을 숨겨도 IconShift는 백그라운드에서 계속 실행됩니다.
- **PWA 복구** — 브라우저 업데이트로 초기화된 아이콘을 포함해 IconShift가 시작될 때 다시 적용합니다.

## 요구 사항

| | 요구 사항 |
| --- | --- |
| macOS | 14 이상 |
| Mac | Apple Silicon 또는 Intel |

IconShift를 사용하는 데 Xcode나 개발자 도구는 필요하지 않습니다.

## 설치

1. [GitHub Releases](https://github.com/kimdaehee0824/IconShift/releases)에서 최신 `IconShift-<버전>.zip`을 받습니다.
2. 압축을 풀고 `IconShift.app`을 응용 프로그램 폴더로 옮깁니다.
3. IconShift를 한 번 엽니다. 현재 릴리스는 ad-hoc 서명을 사용하므로 macOS가 최초 실행을 차단할 수 있습니다. 앱을 열어 본 뒤 **시스템 설정 > 개인정보 보호 및 보안**으로 이동하고 **보안** 영역에서 **그래도 열기**를 누른 다음 다시 **열기**를 확인합니다.
4. IconShift가 처음으로 앱 아이콘을 바꿀 때 **앱 관리** 접근을 허용합니다. 알림을 놓쳤다면 **시스템 설정 > 개인정보 보호 및 보안 > 앱 관리**에서 IconShift를 켭니다.

Developer ID 서명과 노터라이즈가 적용되기 전까지는 IconShift를 업데이트한 뒤 보안 또는 앱 관리 승인을 다시 해야 할 수 있습니다.

## 사용법

1. IconShift를 열고 사이드바에서 **앱 추가**를 누릅니다.
2. 설치된 앱을 고른 뒤 **라이트 아이콘**과 **다크 아이콘** 칸에 이미지를 놓습니다.
3. 자동 전환에는 **시스템 외형 따르기**를 켜 둡니다. 끄면 **항상 라이트** 또는 **항상 다크**를 선택할 수 있습니다.
4. 선택한 아이콘을 바로 다시 적용하려면 **지금 적용**을 누릅니다.

**설정 > 일반**에서 **로그인 시 실행**과 **메뉴 막대 아이콘 표시**를 바꿀 수 있습니다. 메뉴 막대 아이콘을 숨긴 뒤 창을 다시 열고 싶다면 IconShift를 한 번 더 실행하세요. **설정 > 정보**에서는 설치된 버전과 라이선스를 확인할 수 있습니다.

Finder 아이콘은 즉시 바뀝니다. 대상 앱이 이미 실행 중이라면 기존 Dock 아이콘은 그 앱을 종료하고 다시 열 때까지 유지되며, 이는 macOS의 정상 동작입니다. IconShift는 시작될 때 설정된 아이콘을 다시 적용하므로 Safari나 Chrome 업데이트 뒤 PWA 아이콘을 복구하는 데도 도움이 됩니다.
